# Plus Ultra — Userflow Screenshots

Generated by Playwright from the running Vite dev server.
Viewport: **390×844** (iPhone 14 Pro), Chromium.

## How to generate

```bash
# Terminal 1 — start the dev server
cd packages/app
npm run dev

# Terminal 2 — run the capture script
cd packages/app
npx playwright test tests/capture-screens.spec.ts --reporter=list
```

Screenshots land in this directory as `[flow-folder]/step-01.png`, `step-02.png`, …

## Manual capture notes

Each manual folder contains a `MANUAL.md` explaining exactly what state/trigger is needed.

### 40 — Respond to roll call
Active muster must be running (flow 39 completed by another user). Shows the response form with muster details.

### 43 — Receive emergency alert
Full-screen overlay appears when another tribe member broadcasts an emergency alert.

### 45 — Activate bug-out plan (modal)
The modal appears when tapping "Activate" on an existing plan. `step-01.png` shows the empty/create state.

### 48 — Message in federation channel
Requires a successfully federated allied tribe. Route: `/tribe/$tribeId/federation/$channelId`

### 55–57 — Native mesh features
These are Tauri-native: mDNS relay (desktop), WiFi Direct (Android), BLE (mobile). They cannot be emulated in a browser context.

## Notes on screenshot content

Most screens show **empty states** (new install, no data). This is intentional — the designer sees all components, navigation, and layout without needing real data to be meaningful. The Grid Down flows (52–54, 58) inject a `plusultra:offlineSince` timestamp via localStorage before each navigation to simulate each escalation stage.
